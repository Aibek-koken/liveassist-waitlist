# DESIGN.md — LiveAssist AI · Waitlist

Дизайн-направление, референсы и система токенов для короткого презентационного
waitlist-сайта LiveAssist AI. Этот файл — источник правды по визуальному языку.
Если код и этот файл расходятся — приведи их в соответствие.

---

## 1. Направление

**Арт-дирекшен:** _Editorial sales workflow meets calm product intelligence._

Сайт должен читаться как спокойный, серьёзный рабочий продукт, а не как очередной
тёмный AI-SaaS. Ощущение: редакционный разворот о работе менеджера, где данные
разговора аккуратно превращаются в структурированную работу.

Три опорных решения:

1. **Тёплая бумага, а не тёмный экран.** Светлый off-white фон, графитовый текст,
   один яркий акцент. Глубина создаётся hairline-линиями и мягкими тенями, а не
   стеклом и градиентными blob-ами.
2. **Одна связная история, а не сетка карточек.** Центральный объект продукта —
   заметка о разговоре (Session Note). Всё остальное расходится от неё тонкими
   соединительными линиями.
3. **Анимация объясняет продукт.** Главная идея движения — _conversation data
   becomes structured work_. Движение сдержанное, повторяется мягко, полностью
   отключается при `prefers-reduced-motion`.

## 2. Референсы (что изучено и что берём)

Из-за защиты recent.design (HTTP 403 на прямой fetch) насмотренность собрана по
живым продуктовым сайтам того же класса + актуальным трендам 2025–2026
(neo-editorial SaaS, «tech editorial», restrained cinematic motion).

| # | Референс | Что берём | Что сознательно НЕ берём |
|---|----------|-----------|--------------------------|
| 1 | **Attio** (attio.com) — CRM | Идею «данные как связанные узлы», спокойные продуктовые mockup-ы, hairline-линии между сущностями, светлый фон | Их плотный enterprise-грид фич и обилие мелких UI-скриншотов |
| 2 | **Linear** (linear.app) | Ритм секций, крупную типографику, сдержанный reveal при скролле, дисциплину в отступах | Тёмную «инженерную» палитру и глянцевые градиенты |
| 3 | **Anthropic** (claude.com/Anthropic) | Тёплый cream/paper фон, редакционную типографику, generous whitespace, motion как сигнал сдержанности | Serif-дисплей (у нас grotesk) и «исследовательский» тон |
| 4 | **Cal.com / Vercel (Geist)** | Моно для метаданных и статусов, чистые границы, честные product-панели без «фантастики» | Полностью монохром без акцента |
| 5 | **Stripe** (product demo внутри маркетинга) | Приём «живого» встроенного продукта, пошаговую смену состояний как доказательство, а не декор | Тяжёлый параллакс и scroll-jacking |

**Общий вывод для LiveAssist:** редакционная сетка + одна центральная
workflow-визуализация + один встроенный интерактивный демо-модуль объясняют
продукт быстрее, чем список фич. Это ровно то, что нужно для холодного outreach:
за 30–60 секунд показать, что продукт живёт вокруг звонка и заполняет CRM.

Что **не копируем ни у кого**: не собираем страницу из готовых компонентов
21st.dev, не берём чужой лендинг целиком, не используем карусель логотипов,
dashboard-скриншот в рамке браузера, фиолетово-синие AI-градиенты и glowing orbs.

## 3. Цвет

Палитра: тёплая бумага + графит + один электрический lime-акцент + тёмный
success-зелёный. Цвет WhatsApp живёт только внутри WhatsApp-сцен.

| Токен | HEX | Роль |
|-------|-----|------|
| `--paper` | `#F6F4EE` | Основной фон страницы (тёплый off-white) |
| `--milk` | `#FBFAF6` | Приподнятые зоны, чередование секций |
| `--surface` | `#FFFFFF` | Панели, product mockup |
| `--ink` | `#17130E`… используем `#1A1B17` | Основной графитовый текст / заголовки |
| `--ink-soft` | `#6B675E` | Вторичный текст, подписи |
| `--line` | `rgba(26,24,20,0.10)` | Hairline-границы и соединительные линии |
| `--lime` | `#C6F24E` | Акцент: заливка CTA, ключевые метки, живые точки |
| `--lime-deep` | `#9BD11E` | Границы/hover для lime-поверхностей |
| `--success` | `#1F7A46` | Статус «Добавлено», подтверждение записи в CRM |
| `--success-soft` | `rgba(31,122,70,0.10)` | Фон success-строк |
| `--whatsapp` | `#25D366` | Только внутри WhatsApp-сцены (Блок 2, направление 3) |

Правила контраста: lime — **светлый** акцент, поэтому текст на lime всегда
графитовый (`--ink`), а не белый. Lime не используется как крупный фон текста,
который нужно читать долго. Статусы дублируются иконкой + текстом, не только
цветом (WCAG AA, «не полагайся только на цвет»).

## 4. Типографика

Максимум два загружаемых семейства + системный моно для метаданных.

- **Display / заголовки:** `Manrope` (700/800) через `next/font/google`. Крупный,
  редакционный grotesk с выразительным контрастом масштаба.
- **UI / body:** `Inter` (400/500/600) через `next/font/google`.
- **Метаданные, статусы, числа Session Note:** системный моно-стек
  (`ui-monospace, "SF Mono", "JetBrains Mono", Menlo, monospace`) — не добавляет
  третий загружаемый шрифт, но даёт «продуктовый» tech-акцент.

Масштаб (clamp, fluid):

- Hero H1: `clamp(40px, 6vw, 76px)`, `letter-spacing: -0.035em`, `line-height: 1.02`.
- Section H2: `clamp(30px, 4vw, 52px)`, `-0.03em`.
- Lead / подзаголовок: `clamp(16px, 1.4vw, 19px)`, `line-height: 1.6`.
- Body: `15–16px`, `line-height: 1.6`.
- Eyebrow / mono-метки: `12–13px`, `uppercase`, `letter-spacing: 0.16em`.

## 5. Layout

- Контентная ширина: `max-width: min(1180px, 100%)`, боковые поля
  `clamp(20px, 5vw, 40px)`.
- Вертикальный ритм секций: `padding-block: clamp(72px, 12vh, 132px)`.
- Чередование фона: `paper` → `milk` → `paper` → `milk`, чтобы блоки читались
  как отдельные развороты без тяжёлых рамок.
- Радиусы — единая шкала: `12 / 16 / 20 / 26px`. Никаких случайных «pill на всём».
- Тени мягкие и низкоконтрастные: `0 24px 60px -40px rgba(26,24,20,0.28)`.
- Соединительность через `1px` линии и SVG-паттерн, а не через box-ы.

Четыре смысловых блока (+ компактные header/footer):

1. **Hero** — за 5 секунд объясняет продукт; product scene: разговор → overlay →
   Session Note → Suggested Actions → карточка amoCRM.
2. **SessionNoteFlow** — одна композиция: разговор → Session Note (ядро) → три
   направления (CRM сейчас · Скрипты «в разработке» · WhatsApp-агент «в разработке»).
3. **CrmSuggestedActionsDemo** — крупный интерактивный mockup: Session Note +
   Suggested Actions + human review + подтверждение + запись в amoCRM + duplicate skip.
4. **WaitlistForm** — форма раннего доступа с loading / success / error.

## 6. Motion-система

Главная концепция: **conversation data becomes structured work.**

Тайминги:

- Hover / focus: `140ms`.
- Малые UI-переходы: `220ms`.
- Section reveal: `500–650ms`, `ease-out`.
- Product sequence: `700–1100ms` на смысловой шаг.
- Easing: `cubic-bezier(0.22, 1, 0.36, 1)` (спокойный ease-out, без bounce).

Где что:

- **Hero (framer-motion, 9–11s loop):** спокойная waveform → короткие фразы
  выделяются → собираются в Session Note → из неё появляются Suggested Actions →
  одна action связывается линией с карточкой amoCRM. Цикл повторяется мягким
  crossfade, без резкого restart.
- **SessionNoteFlow:** один scroll-linked reveal — при входе секции линии от ядра
  к трём направлениям «прочерчиваются» (stroke-dashoffset), карточки направлений
  всплывают со stagger. Без scroll-jacking, обычная прокрутка не блокируется.
- **CrmSuggestedActionsDemo:** actions появляются последовательно; по нажатию
  «Подтвердить и записать в amoCRM» строки по очереди получают статус «Добавлено»,
  дубль — «Пропущено: уже существует», затем спокойное итоговое подтверждение.
  Только opacity/transform/layout — без анимации blur и тяжёлых теней.

`prefers-reduced-motion: reduce`:

- Бесконечные циклы выключены, показывается финальное осмысленное состояние
  (Session Note собрана, actions «Добавлено», линии прочерчены).
- Контент не скрывается, смысл визуализации сохраняется.

## 7. Почему эта система объясняет LiveAssist

Тёплая бумага снимает «AI-slop» ощущение и говорит «серьёзный рабочий
инструмент». Единственная центральная визуализация (Session Note как ядро)
буквально показывает продуктовую модель: один разговор становится памятью и
расходится в три направления. Встроенный интерактивный демо-модуль — это не
скриншот, а само доказательство: менеджер видит и подтверждает каждое действие,
CRM заполняется, дубль пропускается. Lime-акцент даёт одну точку энергии и
собирает все CTA к одной цели — заявке на ранний доступ.
